###  [Blog 1 - Amazon CloudWatch](3.1-Blog1/)
This blog introduces how to use Amazon CloudWatch to build a proactive monitoring system on AWS. You will learn how CloudWatch collects metrics, logs, and events from AWS services, and how to combine Dashboards, Alarms, and Amazon SNS to detect issues early and automate responses when system anomalies occur. The article also shares practical insights into designing an effective monitoring strategy that improves system availability, reduces incident response time, and enhances operational efficiency in AWS environments.

###  [Blog 2 - AWS Batch & Amazon ECS](3.2-Blog2/)
This blog introduces an architecture for automated extraction of compressed files stored in Amazon S3 by leveraging AWS Batch and Amazon ECS within an event-driven architecture. You will learn how Amazon S3, Amazon EventBridge, AWS Batch, and Amazon ECS work together to build a fully automated data processing pipeline that provisions compute resources only when new files require processing. The article also explains the role of each AWS service, the end-to-end workflow, and why AWS Batch is a more suitable solution than AWS Lambda for large-scale batch processing workloads.

###  [Blog 3 - Security Response Automation](3.3-Blog3/)
This blog introduces how to build an automated security incident response workflow on AWS by integrating services such as AWS CloudTrail, Amazon EventBridge, AWS Lambda, AWS Systems Manager, and Amazon SNS. You will learn how an event-driven architecture enables the rapid detection of security events, automatically executes remediation actions, and notifies the operations team when incidents occur. The article also discusses the benefits of automating security response processes, including reduced response time, improved security posture, and more efficient operational management across AWS environments.
